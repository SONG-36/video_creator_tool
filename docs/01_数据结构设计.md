

# 01_数据结构设计

# 1. 文档目的

本文档定义 Video Creator Tool 核心业务对象的数据结构。

数据结构设计依据：

- Master Design 中定义的业务模块
- 00_业务流程与状态流转设计中定义的状态流转

系统核心原则：

对象负责承载业务数据，状态负责驱动流程流转。

# 2. 核心对象关系

整体关系：

Project
 ↓
Script
 ↓
Storyboard
 ↓
Shot
 ↓
Production Task
 ↓
Asset
 ↓
Generation Result

其中：

- Project 是一次视频生产任务
- Script 是人工输入剧本
- Storyboard 是AI拆分后的镜头集合
- Shot 是最小生产单元
- Production Task 是AI生产任务
- Asset 是生成所需素材
- Generation Result 是最终输出

# 3. Project 数据结构

## 定义

代表一个完整的视频生产项目。

```json
{
  "project_id": "",
  "name": "",
  "product_name": "",
  "video_goal": "",
  "status": "created",
  "created_at": "",
  "updated_at": ""
}
```

## 状态

```text
created
script_input
storyboard_generation
production
completed
```

# 4. Script 数据结构

## 定义

保存人工输入的视频剧本。

```json
{
  "script_id": "",
  "project_id": "",
  "content": "",
  "version": 1,
  "status": "draft",
  "created_by": "",
  "created_at": ""
}
```

## 状态

```text
draft
submitted
approved
rejected
```

# 5. Storyboard 数据结构

## 定义

保存AI根据剧本生成的结构化分镜结果。

```json
{
  "storyboard_id": "",
  "script_id": "",
  "shots": [],
  "version": 1,
  "status": "draft"
}
```

# 6. Shot 数据结构（核心对象）

## 定义

Shot代表一个独立视频镜头，是系统最小生产单位。

```json
{
  "shot_id": "",
  "storyboard_id": "",
  "shot_number": 1,

  "time_range": {
    "start": 0,
    "end": 5
  },

  "scene": "",
  "purpose": "",
  "action": "",
  "camera": "",

  "production_type": "pending",

  "review_status": "waiting_review"
}
```

## production_type

```text
pending
real_shoot
ai_generate
```

## review_status

```text
waiting_review
approved
revision_required
rejected
```

# 7. Shot Review 数据结构

## 定义

保存人工审核记录，不直接覆盖Shot数据。

```json
{
  "review_id": "",
  "shot_id": "",
  "review_type": "storyboard",
  "result": "approved",
  "comment": "",
  "reviewer": "",
  "created_at": ""
}
```

## result

```text
approved
revision_required
rejected
```

```json

  "result_id": "",

  "video_path": "",
  "generation_cost": 0,
  "status": "completed",
  "review_status": "reviewing",
  "created_at": ""
}
```

## review_status

```text
reviewing
approved
rejected
regenerate
```

# 11. 数据追踪原则

所有生产过程必须保留历史：

- 原始剧本
- 分镜版本
- 审核记录
- Prompt版本
- 素材版本
- 生成记录
- 视频结果

禁止直接覆盖历史数据。

# 12. MVP数据范围

V0.1必须支持：

- Project
- Script
- Storyboard
- Shot
- Shot Review
- Production Task
- Asset

V0.2增加：

- Generation Result
- 成本统计
- 视频版本管理

暂不设计：

- 多用户权限
- 复杂审批流
- 自动评分系统
- 商业决策数据模型
