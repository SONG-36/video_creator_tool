

# 02_页面设计

# 1. 页面设计目标

本页面设计基于：

- Master Design 定义的业务流程
- 00_业务流程与状态流转设计定义的人工审核节点
- 01_数据结构设计定义的数据对象

页面目标：

让用户能够清晰完成：

剧本输入 → 分镜审核 → 生产决策 → AI素材准备 → 视频生成 → 结果审核

整个生产流程。

# 2. 产品设计原则

## 2.1 工作流优先

页面不是传统后台管理系统，而是视频生产工作台。

用户核心任务：

完成一个视频任务。

因此采用：

项目中心 + 工作流步骤

设计。

## 2.2 状态可视化

所有页面必须显示当前状态：

例如：

- 待审核
- 等待素材
- 生成中
- 已完成

避免用户不知道系统当前进度。

## 2.3 人机协作

AI负责生成建议。

人工负责确认。

所有关键节点必须有明确操作按钮。

# 7. Storyboard分镜审核页面

## 核心页面

这是整个系统最重要页面。

## 页面布局

采用卡片式镜头列表。

每个Shot一个卡片。

示例：

```text
--------------------------------
Shot 001

时间：0-3s

场景：
汽车座椅污渍特写

目的：
制造视觉冲击

动作：
展示污渍

AI建议：
AI生成

状态：待审核

[通过] [修改] [驳回]
--------------------------------
```

## 卡片展开后显示

- Camera
- Action
- AI建议
- 生产方式

# 13. UI视觉设计规范

## 整体风格

定位：

专业AI视频生产工具。

风格：

- 简洁
- 现代
- 工具感
- 高信息密度

参考方向：

AI创作平台 + 专业剪辑软件。

# 14. 色彩设计

基础：

- 白色背景
- 灰色辅助区域
- 深色文字

状态颜色：

成功：绿色

处理中：蓝色

警告：黄色

失败：红色

# 15. 交互设计原则

## 原则1

重要操作必须明确按钮。

例如：

通过
驳回
重新生成

## 原则2

减少页面跳转。

尽量在工作台完成操作。

## 原则3

所有AI结果允许人工修改。
